
New concept for 1.0: everything in objects, mainly PChars instead of strings

Modules:

FileBase
- unit
- files.bbs
- files.lng
- files.idx (dates for NewFilesHatch)
- (nearly) unlimited description length
- using charset conversion
- Touch, ArcTouch
- reformat descriptions

Cfg
- unit
- use fidoconfig (statically linked under DOS)
- use inline-functions for using Cfg

Log
- unit
- only binkley-style

Outbound
- unit
- supported outbounds:
  - Binkley Domain
  - new FD
  - T-Mail fileboxes
- interface:
  - CheckBusy, SetBusy, UnSetBusy
  - SendFile, CheckFileSent
  - ArchiveName (BT: in outbound, TMail: in filebox, FD: subdirs in TicDir)
  - PurgeArchs (remove archives with zero length)

MsgBase
- unit
- using smapi (statically linked under DOS)

Dupes
- unit
- add Dupe-Entry
- check for Dupe
- purge Dupes

Announce
- unit
- add File
  - AddFile(Area: PArea; Name: PChar; Desc: PChar; From: TNetAddr)
- write Announces
  - WriteAnnounces
- NetMail-Announces
- EchoMail-Announces
  - Announce-Groups
- using Templates (MsgHeader, MsgFooter, GroupHeader, GroupFooter, File)
- using charset conversion
- reformat descriptions

TICFile
- unit
- ADT (Abstract Data Type) Tic
  - Read
  - Write
  - Get/Set
  - Clone

Chrs
- unit
- charset conversion

NLS
- unit
- National Language Support
- formats like "argument 1: %1, argument 3: %3, argument 2: %2\n"

Template
- unit
- template parser
  - Input: Filename, Array of PChars with Variable-Names and Values
  - Output: PChar with whole text

Misc
- unit
- check busy file
- MsgID
- append error message to TIC
  - AppendErrorStrTIC(ErrNum: Byte)
- create newareas.pt, areas.log
  - AddAutoCreatedArea(name: string)
  - AddProcessedArea(name: string)
  - WriteAutoCreatedAreas
  - WriteProcessedAreas
  - using PChar-B-Tree
- Addr2Str
- Str2Addr
- CompAddr
- CopyAddr
- write netmail on errors
- some global types

BTree
- unit
- generic B-Tree
- PChar-B-Tree
- Address+Pointer-B-Tree

PTToss
- program
- ScanForTICs
- read TIC (using unit TIC)
- CheckTIC
  - CheckSize
- AutoCreate
  - areaname masks like "BBS-*" for all BBSNet-Areas
    (<one AKA, multiple groups>-problem)
- ProcessTIC
- ForwardTIC
  - AddToArcList
    - using Address+Pointer-B-Tree
- Extract and use FILE_ID.DIZ / LSM, order configurable
- write magics from TIC to a configurable file (add/replace appropriate line)
- execute script after single files / each file

PTFFix (FileFix)
- program
- forward requests to uplinks
- %From
- remote maintenance
- %Info
- %Notify
- all responses in one msg
- response templates
- ReScan

PTNFH (NewFilesHatch)
- program
- Exclude

PTUnPack
- program
- look for archives in inbound and unpack them

PTPack
- program
- pack TICs and files in outbound
  - AllFix-style
  - FileScan-style
  - MTic-style
- readArcList

PTHatch
- program
- command-line
- ncurses-interface

PTMaint
- program
- remove obsolete passthrough-files
- purge Dupes (using unit Dupes)
- purge old files from filebase
- remove old archives from outbound

PTFFind (FileFind)
- answer to echomail search requests like AllFix

ProTick
- wrapper for UnPack, Toss, Scan, Hatch, NFH, Maint, Pack



No more RegKeys, release under GPL.
Not using GeneralP.pas and Types.pas anymore.
Generate .deb- and .rpm- Binary(elf-i386-glibc2)/Source-Packages.
